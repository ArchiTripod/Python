包括远程命令ssh,不需要另外使用第三方ssh库如paramiko.

几个关键点:
1.pexpect.run() 代替 os.system() 前者可以程序内获得系统输出值进行处理
2.pexpect.spawn('ssh [user@]hostname [command]') 可以在本地 让远程 hostname 机器
command命令并获取结果 (可用 && 连接多个命令).
3.通过命令'bash -c "command"' 执行含有 重定向,管道和通配符的命令
如: pexpect.run('bash -c "ps -ef | grep -e 'ssh.*-D'")

ipython notebook + Matplotlib 终于写出了不在终端下的非input output的程序。

事情起因与学校计算方法的上机实验。

首先ipython notebook提供了强大的交互式环境，并且支持各种magic函数：
  • !cmd :直接运行系统命令。（可能不归在magic中，magic是以%开头）
  • %load url ： 导入web或者本地文件
  • %run ： 运行py文件
  • %timeit ： 性能测试
更多强大功能请自己发觉。
notebook不仅仅支持python，还支持makedown，可以边写程序边做笔记
notebook 同时内置了matplotlib backend，可以直接在notebook里面用matplotlib画图，图文程序并茂，写论文形式的文章利器。
这就引入了matplotlib这个大杀器，免费而且功能强大，实现了大部分matlab的功能（之前表述的不正确，通常指得是scipy这一套科学计算库），甚至连函数名称都是相似的（方便转型）。
matplotlib在python环境下，比较为现代大学生所接受，不像matlab，身边的同学都很排斥。同时，可以结合更多的python库来做更多的事情，而不是单单matlab的计算。
matplotlib很多功能没有使用过，但是作为一个计算方法实验，能十分清晰的展示出各种计算方式的图形，与真实函数的差距，并且短短几行代码就可以实现，深深为此折服。

import scipy
import numpy
from scipy import interpolate

def itp(startx, endx, rstf, x, inputx):
    def X(n):
        x = []
        for k in range(n + 1):
            x.append(startx + (endx - startx) * float(k) / n)
        return x


    x = X(x)
    print x

    y = map(rstf, x)
    print y

    p = interpolate.lagrange(x, y)
    print numpy.poly1d(p)

    print map(lambda x : '%.3f' %x, map(rstf, inputx))
    print map(lambda x : '%.3f' %x, map(p, inputx))
    

    pylab.plot(numpy.linspace(startx, endx, 100), map(p, numpy.linspace(startx, endx, 100)), '-')
    pylab.plot(numpy.linspace(startx, endx, 100), map(rstf, numpy.linspace(startx, endx, 100)), '-')
    pylab.show()
